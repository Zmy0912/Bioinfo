# BED/GFF3文件坐标转换工具 - 使用说明

## 功能说明

本工具支持两种格式文件的坐标转换：
- **BED格式**: 将BED文件中每个基因内部的坐标转换为从0开始的相对坐标
- **GFF3格式**: 将GFF3文件中每个基因内部的坐标转换为从0开始的相对坐标

转换后保持基因内部各元素之间的相对距离不变。

## 使用步骤

### 通用步骤

1. **启动程序**
   - 双击运行 `bed_converter.py` 文件
   - 或在命令行执行: `python bed_converter.py`

2. **选择文件类型**
   - 在界面上方选择"BED格式"或"GFF3格式"

3. **选择输入文件**
   - 点击"浏览"按钮选择要转换的文件
   - 或直接在输入框中输入文件路径

4. **加载文件**
   - 点击"加载文件"按钮查看原始文件内容
   - 文件内容会显示在预览区域

5. **转换坐标**
   - 点击"转换"按钮开始转换
   - 程序会自动识别基因ID，并将每个基因内的所有特征坐标转换为相对于该基因起点的坐标
   - 转换结果会实时显示在预览区域

6. **保存文件**
   - 点击"保存文件"按钮保存转换后的结果
   - 选择保存位置和文件名
   - BED格式默认文件名为"基因成员_转换后.bed"
   - GFF3格式默认文件名为"转换后.gff3"

7. **清空**
   - 点击"清空"按钮清除所有数据，重新开始

## BED文件格式说明

输入BED文件格式：
```
geneID  start  end  featureType  phase(optional)
```

- 第1列: 基因/转录本ID
- 第2列: 起始位置
- 第3列: 结束位置
- 第4列: 特征类型 (如 exon, CDS, gene, mRNA等)
- 第5列: 相位 (可选，0, 1, 2 或 .)

## GFF3文件格式说明

输入GFF3文件格式（标准GFF3格式）：
```
seqid  source  type  start  end  score  strand  phase  attributes
```

- 第1列: 序列ID
- 第2列: 来源
- 第3列: 特征类型 (gene, mRNA, exon, CDS等)
- 第4列: 起始位置 (1-based)
- 第5列: 结束位置
- 第6列: 得分
- 第7列: 链方向 (+, -, .)
- 第8列: 相位 (0, 1, 2, .)
- 第9列: 属性 (键值对，用;分隔)

程序会从attributes字段中提取gene_id来识别基因。

## 转换原理

对于每个基因：
1. 找到该基因的最小起始位置 (gene_start)
2. 将该基因内所有特征的坐标减去 gene_start
3. 保持特征之间的相对距离不变

### BED格式示例

```
原始:
AoNCED1  61386543  61389501  gene
AoNCED1  61388987  61389501  exon
AoNCED1  61387535  61387538  exon

转换后:
AoNCED1  0  2958  gene
AoNCED1  2444  2958  exon
AoNCED1  992  995  exon
```

### GFF3格式示例

```
原始:
chr1  Ensembl  gene  61386544  61389501  .  +  .  gene_id=AoNCED1
chr1  Ensembl  exon  61388988  61389501  .  +  .  gene_id=AoNCED1
chr1  Ensembl  exon  61387536  61387539  .  +  .  gene_id=AoNCED1

转换后 (保持GFF3格式):
chr1  Ensembl  gene  1  2958  .  +  .  gene_id=AoNCED1
chr1  Ensembl  exon  2445  2958  .  +  .  gene_id=AoNCED1
chr1  Ensembl  exon  993  996  .  +  .  gene_id=AoNCED1
```

## 注意事项

### 通用注意事项
- 程序会保留原始文件中的注释行（以#开头的行）
- 同一个基因的所有特征会被一起处理
- 转换前请先预览原始文件，确保格式正确
- 建议先查看转换结果预览，确认无误后再保存文件

### BED格式注意事项
- BED格式使用0-based坐标系统
- 转换后坐标直接从0开始

### GFF3格式注意事项
- GFF3使用1-based坐标系统
- 转换后的文件仍保持GFF3格式和1-based坐标系统
- 确保GFF3文件中attributes字段包含gene_id属性
- 程序会保留GFF3文件的所有原始结构、注释行和属性

## 运行环境

- Python 3.x
- 需要安装 tkinter (通常随Python一起安装)

## 常见问题

**Q: 为什么GFF3转换后还是1-based坐标？**
A: GFF3标准使用1-based坐标系统。程序内部计算时转为0-based，但输出时转换回标准的1-based格式以确保兼容性。

**Q: 我的GFF3文件转换后没有数据？**
A: 请检查文件中是否包含gene_id属性。程序通过gene_id识别基因。如果使用其他属性名，可能需要调整程序。

**Q: 可以同时处理多个文件吗？**
A: 当前版本一次只能处理一个文件。处理完一个文件后，点击"清空"再处理下一个文件。

如遇到其他问题，请检查输入文件格式是否符合要求。
