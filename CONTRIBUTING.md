# Contributing Guide

Thank you for your interest in this project! We welcome contributions of any kind.

## How to Contribute

### Report Bugs

If you find a bug or have a feature suggestion:

1. Check the [Issues](https://github.com/Zmy0912/Bioinfo/issues) list to make sure the issue hasn't been reported
2. If there's no related issue, create a new one
3. In the issue, clearly describe:
   - The problem you encountered or the feature you suggest
   - Steps to reproduce the issue (for bugs)
   - Expected behavior
   - Actual behavior (for bugs)
   - Screenshots or error logs (if available)

### Submit Code

#### 1. Fork the Repository

Click the Fork button in the top right corner to copy the repository to your GitHub account.

#### 2. Clone the Repository

```bash
git clone https://github.com/Zmy0912/Bioinfo.git
cd Bioinfo
```

#### 3. Create a Branch

Create a new branch for your changes:

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

#### 4. Make Changes

- Follow the project's code style conventions
- Ensure code is clear and readable
- Add necessary comments
- Write tests for new features

#### 5. Commit Changes

```bash
git add .
git commit -m "Describe your changes"
```

**Commit Message Guidelines:**
- Use clear and concise commit messages
- Short description in first line (under 50 characters)
- Add detailed explanation if needed
- Reference format: `type(scope): description`
  - `feat`: New feature
  - `fix`: Bug fix
  - `docs`: Documentation update
  - `style`: Code formatting
  - `refactor`: Refactoring
  - `test`: Testing related
  - `chore`: Build or tooling

#### 6. Push to GitHub

```bash
git push origin feature/your-feature-name
```

#### 7. Create Pull Request

1. Visit the original repository page
2. Click "New Pull Request"
3. Select your branch
4. Fill in the PR description:
   - Clearly explain what you changed
   - Link related issues (if any)
   - Add screenshots or demos (if applicable)
5. Submit the PR

## Code Standards

### Code Style

- Follow the existing code style in the project
- Use consistent indentation (recommend spaces, 2 or 4)
- Use meaningful names for variables and functions
- Keep functions short and focused on single responsibility

### Testing

- Add tests for new features
- Ensure all tests pass
- Test coverage should not decrease

### Documentation

- Update relevant documentation
- Add necessary comments
- Update README (if needed)

## Code of Conduct

- Respect all contributors
- Use friendly and inclusive language
- Accept constructive criticism
- Focus on what is best for the community
